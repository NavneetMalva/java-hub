BiConsumer<T, U> represents an operation that accepts two input arguments and returns no result.
@FunctionalInterface
public interface BiConsumer<T, U> {
    void accept(T t, U u);
}
ğŸ‘‰ When do we use it?
When you want to perform an action using two values
Commonly used in Maps, Streams, logging, updating objects

BiConsumer<String, String> bc2 = (a, b) -> System.out.println(a.toUpperCase());

7ï¸âƒ£ LiveLock
ğŸ‘‰ What is LiveLock?
Threads are active but never progress because they keep responding to each other.
ğŸ”¹ Example
class Worker {
    private boolean active = true;

    synchronized void work(Worker other) {
        while (active) {
            if (other.active) {
                continue; // keeps yielding
            }
            active = false;
        }
    }
}

ğŸ”¹ Difference Summary
Problem	Description
Deadlock	Threads blocked forever
Livelock	Threads active but stuck
Starvation	Thread never gets CPU
